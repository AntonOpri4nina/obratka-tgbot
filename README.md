# 📨 Telegram Feedback Bot

Бот для обратной связи: клиент пишет в личку → сообщения попадают в топик супергруппы → менеджер отвечает → ответ летит клиенту обратно.

```
Клиент (личка)  ←──────────────────────→  Топик в супергруппе
    │  "Здравствуйте, у меня вопрос..."        │  👤 Иван Иванов · 123456
    │  ✅ Сообщение получено!                   │  [Forwarded] "у меня вопрос..."
    │                                           │
    │  ← "Здравствуйте! Отвечаем..."            │  Менеджер: "Здравствуйте! Отвечаем..."
```

---

## ⚙️ Подготовка

### 1. Создайте бота
1. Напишите [@BotFather](https://t.me/BotFather) → `/newbot`
2. Скопируйте **токен** (вида `1234567890:AAxxxx...`)

### 2. Создайте супергруппу с топиками
1. Создайте обычную группу в Telegram
2. **Настройки → Тип группы → Супергруппа** *(либо просто создайте сразу публичную/приватную супергруппу)*
3. **Настройки → Темы** → включите **Темы** (Forum/Topics)
4. Добавьте бота в группу и выдайте права **Администратора** с разрешениями:
   - ✅ Управление темами (*Manage Topics*)
   - ✅ Отправка сообщений
   - ✅ Закрепление сообщений *(для карточки пользователя)*

### 3. Узнайте ID группы
Самый простой способ — перешлите любое сообщение из группы боту
[@getmyid_bot](https://t.me/getmyid_bot) или [@userinfobot](https://t.me/userinfobot).  
ID будет выглядеть как `-1001234567890` (начинается с `-100`).

## 🚀 Запуск (Python напрямую)

```bash
# 1. Клонируйте / скопируйте файлы в папку
cd feedback_bot

# 2. Установите зависимости
pip install -r requirements.txt

# 3. Создайте файл с настройками
cp env.example .env
# Откройте .env и вставьте BOT_TOKEN и GROUP_CHAT_ID

# 4. Запустите
python bot.py
```

---

## 🐳 Запуск через Docker (рекомендуется для сервера)

```bash
cp env.example .env
# Отредактируйте .env

docker compose up -d          # запуск в фоне
docker compose logs -f        # просмотр логов
docker compose down           # остановка
```

Данные (`users.json`) хранятся в папке `./data/` и не теряются при перезапуске контейнера.

## 💬 Как это работает

| Событие | Что происходит |
|---|---|
| Клиент нажимает `/start` | Бот отвечает: *"Здравствуйте, Имя Фамилия! Напишите вопрос..."* |
| Клиент пишет первое сообщение | Создаётся топик `"Имя Фамилия · ID"`, сообщение пересылается туда |
| Клиент пишет повторно | Сообщение пересылается в уже существующий топик |
| Менеджер отвечает в топике | Ответ мгновенно доставляется клиенту в личку |
| Клиент запрещает пересылку | Бот использует `copy_message` вместо `forward` (без имени) |

## ❓ Частые вопросы

**Топик случайно удалили** — клиент сможет снова написать, и создастся новый топик.  
**Бот не создаёт топики** — проверьте, что у бота есть право *Manage Topics* в группе.  
**Не получает сообщения из группы** — убедитесь, что `GROUP_CHAT_ID` указан с минусом: `-100...`.  
**Несколько ботов** — каждый бот работает независимо, просто создайте несколько `.env` / контейнеров.
